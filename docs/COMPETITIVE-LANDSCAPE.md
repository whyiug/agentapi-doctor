# Related-project comparison guidelines

Comparisons with other projects must use current primary sources and the same
dimensions: capability overlap, architecture, evidence model, client-observed
behavior, governance, reuse opportunity, and whether a separate tool is useful.

A comparison should record the source URL, retrieval date, relevant
commit/release, license and reuse status, and uncertainty. It should say when
another project is the better choice and must not infer endorsement or
willingness to collaborate from silence.

No outreach, adoption, review, or external endorsement is claimed unless it
actually occurred and the non-sensitive evidence can be cited.
